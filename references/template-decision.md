# 模板决策

目标：不要从几套固定模板里机械选择，而是先生成 `design_brief`，再组合版式语法。

## 决策输入

每次从 JD、公司和用户材料中判断：行业、岗位、公司气质、用户材料强弱、投递场景。

## 输出 design_brief

必须先产出这个结构，再生成 HTML：

```json
{
  "tone": "亲和、可信赖、基层管理潜力",
  "layout": "sidebar-card",
  "palette": {
    "primary": "#8f1f17",
    "accent": "#f4bd2a",
    "soft": "#fff2df",
    "paper": "#fffaf2"
  },
  "brand_reason": "好想你品牌为红黄系，使用深红和暖黄作为相邻色，避免广告化高饱和大面积铺色。",
  "photo": { "mode": "ask", "use_if_provided": true, "reason": "门店服务岗位适合职业照" },
  "density": "content-light",
  "module_order": ["header", "profile", "summary_cards", "education", "competency", "certificates"],
  "avoid": ["求职说明", "投递关键词", "匹配策略", "缺口分析"]
}
```

## 版式骨架选择

- `sidebar-card`：左侧信息栏 + 右侧正文。适合门店、销售、管培、行政、通用校招，尤其内容偏少时。
- `top-card`：顶部名片 + 单栏正文。适合经历较完整、ATS 友好和稳重岗位。
- `timeline`：左时间线 + 右成果。适合实习/项目经历较多的学生。
- `portfolio-light`：作品入口 + 视觉模块。适合设计、品牌、内容、新媒体。
- `technical-grid`：技能矩阵 + 项目块。适合研发、数据、工程、产品。

## 内容强弱对应策略

- 经历少：用核心能力卡片、岗位胜任力、教育背景、证书能力撑结构；不要编造经历。
- 经历多：减少口号式摘要，把空间留给项目、实习和量化结果。
- 有照片：门店、销售、服务、行政、运营可用；技术、法务、纯 ATS 默认不用。
- 无照片：不要放空框，改用姓名名片、能力标签和色块建立视觉锚点。

## 禁止

不要把设计决策、JD 分析、风险提示写进简历正文。标题、标签、meta 小字里也不要写“匹配：”“匹配 JD：”，直接写自然能力词或发展路径。内部词进入简历时必须转译：

| 内部词 | 简历可用表达 |
|---|---|
| 匹配策略 | 培养方向 / 职业目标 |
| 投递关键词 | 核心能力 / 岗位优势 |
| JD 反推 | 岗位理解 |
| 缺口分析 | 面试准备材料，不进简历 |
