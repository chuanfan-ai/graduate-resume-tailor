# 动态设计系统

这不是固定模板库，而是简历版式语法。每份简历根据 `design_brief` 组合生成。

## 设计 token

`design_brief.palette` 控制：

- `primary`：主色，用于头部、侧栏标题、关键线条。
- `accent`：强调色，用于节点、标签、数字、分隔点。
- `soft`：浅背景，用于侧栏、卡片底色。
- `paper`：纸张底色。
- `ink`：正文深色，可省略。
- `muted`：辅助文字，可省略。

品牌色只做相邻设计：可借用 logo 色相，但要降低饱和度或控制面积，避免像广告。

## 版式语法

### sidebar-card
结构：品牌头图 / 左侧信息栏 / 右侧正文。适用：门店、销售、服务、运营、行政、管培、通用校招、经历偏少。

### top-card
结构：顶部名片 / 横向联系信息 / 单栏正文。适用：金融、咨询、国企、工程、法务、医疗、ATS 友好投递。

### timeline
结构：顶部名片 / 时间轴经历 / 技能标签。适用：有 2 段以上实习、项目、竞赛、社团经历。

### portfolio-light
结构：照片或作品入口 / 视觉摘要 / 经历模块。适用：设计、传媒、品牌、新媒体、内容运营。

### technical-grid
结构：技能矩阵 / 项目卡片 / 教育背景。适用：研发、数据、工程、产品。

## 模块库

可组合模块：`profile_panel`、`photo_block`、`summary_cards`、`experience_timeline`、`competency_blocks`、`education_block`、`skills_chips`、`portfolio_links`。

## 质量检查

交付前自检：A4 单页稳定、视觉锚点明确、层级清楚、无内部分析词、照片处理自然、配色匹配公司/行业且不过度广告化、内容少时靠结构解决而不是塞解释性废话。
